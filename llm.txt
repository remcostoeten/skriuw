# Better Auth Implementation Guide

## Overview
This document outlines the steps to implement authentication using `better-auth` in the Skriuw project. The goal is to have a robust authentication system with Email/Password, GitHub OAuth, and Anonymous authentication, featuring a seamless user experience with instant state updates.

## Prerequisites
- `better-auth` package installed.
- PostgreSQL database configured.
- Environment variables set.

## Step-by-Step Implementation

### 1. Installation
Install the necessary packages in the `apps/web` (or relevant) package:
```bash
npm install better-auth
```

### 2. Environment Configuration
Add the following to your `.env` file:
```env
BETTER_AUTH_SECRET=your_generated_secret_key
BETTER_AUTH_URL=http://localhost:3000 # or your app's base URL
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret
```
Generate the secret using `openssl rand -base64 32`.

### 3. Server-Side Configuration (`auth.ts`)
Create `apps/web/lib/auth.ts` (or similar path):
```typescript
import { betterAuth } from "better-auth";
import { drizzleAdapter } from "better-auth/adapters/drizzle";
import { db } from "@/db"; // Import your Drizzle instance
import { anonymous } from "better-auth/plugins";

export const auth = betterAuth({
  database: drizzleAdapter(db, {
    provider: "pg",
  }),
  emailAndPassword: {
    enabled: true,
  },
  socialProviders: {
    github: {
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    },
  },
  plugins: [
    anonymous(),
  ],
});
```

### 4. API Route Handler
Create `apps/web/app/api/auth/[...all]/route.ts`:
```typescript
import { auth } from "@/lib/auth"; // Adjust path as needed
import { toNextJsHandler } from "better-auth/next-js";

export const { GET, POST } = toNextJsHandler(auth);
```

### 5. Client-Side Configuration (`auth-client.ts`)
Create `apps/web/lib/auth-client.ts`:
```typescript
import { createAuthClient } from "better-auth/react";
import { anonymousClient } from "better-auth/client/plugins";

export const authClient = createAuthClient({
  baseURL: process.env.NEXT_PUBLIC_APP_URL || "http://localhost:3000",
  plugins: [
    anonymousClient(),
  ],
});

export const { signIn, signUp, useSession, signOut } = authClient;
```

### 6. Database Schema Update
Run the CLI to generate the schema:
```bash
npx @better-auth/cli generate
```
Then migrate your database using your project's migration tool (e.g., `bun run db:push` or similar).

### 7. UI Implementation
#### Sign In / Sign Up View
Create a component that handles both sign-in and sign-up.
- Use `signIn.email` and `signUp.email`.
- Use `signIn.social({ provider: 'github' })`.
- Use `signIn.anonymous()`.
- Conditionally render GitHub button based on env vars availability (check if `GITHUB_CLIENT_ID` is set).

#### User Avatar Menu
- Use `useSession()` to get the current user.
- If logged in, show the avatar.
- If not, show a "Sign In" button.
- Implement `signOut()` functionality.

### 8. Instant State Update
The `useSession` hook from `better-auth/react` uses `nanostores` to provide reactive state updates. Ensure you use this hook in your components to get instant updates without page reloads.

## Notes
- Ensure `BETTER_AUTH_SECRET` is secure.
- Configure GitHub OAuth app with the correct callback URL: `${BETTER_AUTH_URL}/api/auth/callback/github`.
