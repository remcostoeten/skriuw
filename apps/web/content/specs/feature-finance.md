# Finance Editor Type

## Goal
Track income, expenses, debt, savings and overall financial status inside the application without losing current aesthetic and easy usage of the Block Note editor.
This will require multiple individual components that work together, either pre-inserted in the document or able to be added individually.

## Scope
Upon first time using any finance block, user **MUST** create a wallet or space in which everything gets nested.
This way if you call an expense/goal block in a random note, you need to specify which wallet it belongs to.

## Wallet Setup
- Name
- Type of account
- Current balance (optional)

## Modules
The following modules will be available: **savings**, **expense**, **income**, **debt**.

### Module Entry Structure
-[ ] Create an entry
-[ ] Name
-[ ] Optional description
-[ ] Amount
-[ ] Recurring option
-[ ] Category
-[ ] Total (tlt)
