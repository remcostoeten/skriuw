# Requirements

## System Requirements

### Operating System

Servo works on:
- **Linux** (tested on Debian/Ubuntu) - amd64, arm64
- **macOS** (Darwin) - amd64, arm64
- **Windows** - amd64, arm64

### Runtime Dependencies

**None!** Servo is distributed as a statically-linked binary. No runtime dependencies required.

### Build Dependencies (Optional)

**Only needed if building from source:**

- **Go 1.21 or later** (for compiling)
- Check your version: `go version`

**Go dependencies (automatically fetched):**
- Go standard library
- [Bubble Tea](https://github.com/charmbracelet/bubbletea) v0.25.0
- [Lipgloss](https://github.com/charmbracelet/lipgloss) v0.9.1

All dependencies are managed via `go.mod` and fetched automatically when building.

## Project Structure Requirements

### Supported Project Types

Servo works with:

1. **Monorepos** (recommended)
   - Must have `package.json` at root
   - Must define `workspaces` field
   - Workspace directories: `apps/*`, `packages/*`

2. **Standalone Projects**
   - Must have `package.json` at root
   - Must have a `dev` script in `package.json`

### Package Manager Detection

Servo automatically detects your package manager by checking (in order):

1. `packageManager` field in `package.json`
2. Lock files (priority order):
   - `bun.lockb` or `bun.lock` → bun
   - `pnpm-lock.yaml` → pnpm
   - `yarn.lock` → yarn
   - `package-lock.json` → npm

### App Discovery Requirements

For an app to be discovered, it must:

1. **Be in a workspace directory** (for monorepos)
   - Or be the root directory (for standalone)

2. **Have a `package.json`** with:
   - A `name` field (for display)
   - A `scripts` object
   - A `dev` script (required for "Run" menu)

3. **Optional but recommended:**
   - `build` script (for "Build" menu)
   - `tauri:dev` or `tauri` script (for Tauri variant)
   - `tauri:build` script (for Tauri builds)

### Example Valid Structure

```
my-project/
├── package.json          # Root package.json with workspaces
├── bun.lockb            # Package manager lock file
└── apps/
    ├── web/
    │   └── package.json  # Must have "dev" script
    └── docs/
        └── package.json  # Must have "dev" script
```

### Example package.json

```json
{
  "name": "my-project",
  "workspaces": ["apps/*"],
  "packageManager": "bun@1.1.0",
  "scripts": {
    "dev": "bun run --filter @myproject/web dev"
  }
}
```

### Example App package.json

```json
{
  "name": "@myproject/web",
  "scripts": {
    "dev": "next dev --port 42069",
    "build": "next build",
    "tauri:dev": "tauri dev",
    "tauri:build": "tauri build"
  }
}
```

## Port Requirements

Servo monitors these ports for process management:
- `42069` (default Next.js dev port)
- `6969` (alternative dev port)
- `1420` (alternative dev port)

If your apps use different ports, you can still use Servo, but the "Kill Dev Processes" utility may not detect them.

## Build Requirements

### For Building Apps

- Node.js 18+ (for most JavaScript/TypeScript projects)
- Bun 1.1.0+ (if using bun as package manager)
- Rust toolchain (for Tauri apps)

### For Deploying

- Vercel CLI installed (`npm i -g vercel`)
- Vercel account configured
- Project linked to Vercel

## Terminal Requirements

### Minimum Terminal Size

- Width: 90+ characters (for proper UI rendering)
- Height: 24+ lines (for menu and output display)

### Terminal Features

Servo works best with:
- **True color support** (for best visual experience)
- **Unicode support** (for icons and borders)
- **ANSI escape codes** (standard on modern terminals)

### Recommended Terminals

- **Linux**: Alacritty, Kitty, GNOME Terminal, Konsole
- **macOS**: iTerm2, Terminal.app, Alacritty
- **Windows**: Windows Terminal, Alacritty, WSL terminal

## Optional Tools

These enhance Servo's functionality but aren't required:

- **lsof** - For port detection and process management (Linux/macOS)
- **pkill** - For killing processes by pattern (Linux/macOS)
- **xdg-open** - For opening browser (Linux)
- **open** - For opening browser (macOS)
- **git** - For GitHub repo detection (future feature)

## Verification

Check if your project is compatible:

```bash
# 1. Check Go version
go version  # Should be 1.21+

# 2. Check project structure
ls package.json  # Should exist

# 3. Check for workspaces (if monorepo)
cat package.json | grep workspaces

# 4. Check for apps
ls apps/  # Should exist for monorepos

# 5. Test Servo
cd tools/servo
go run .
```

If Servo doesn't detect your apps, check:
- ✅ `package.json` exists at root
- ✅ Workspaces are defined (for monorepos)
- ✅ App directories have `package.json` with `dev` script
- ✅ Package manager lock file exists
