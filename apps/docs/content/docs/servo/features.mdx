# Features

Servo provides a comprehensive set of features for managing development workflows.

## Core Capabilities

### 🎯 Project Auto-Detection

**Automatic Project Structure Detection**
- Detects monorepo vs standalone projects
- Identifies workspace directories (`apps/*`, `packages/*`)
- Scans for apps with `dev` scripts
- No manual configuration required

**Package Manager Detection**
- Detects bun, pnpm, yarn, or npm
- Checks `packageManager` field first
- Falls back to lock file detection
- Uses correct commands automatically

**App Discovery**
- Finds all runnable apps in workspaces
- Extracts app names from `package.json`
- Handles scoped packages (`@scope/name`)
- Discovers Tauri variants automatically

### 🚀 Run Applications

**Dev Server Management**
- Start any discovered app with one selection
- Live output streaming (last 200 lines)
- Automatic port detection from output
- Real-time status updates

**Interactive Controls While Running**
- `o` - Open browser at detected port
- `r` - Restart server
- `p` - Install package via bun
- `g` - Open GitHub repo
- `q/esc` - Stop and return to menu

**Port Detection**
- Multiple regex patterns for port extraction
- Supports: `localhost:PORT`, `http://...:PORT`, `port PORT`
- Auto-opens browser when port detected
- Shows port in status display

### 🔨 Build Applications

**Build Management**
- Build any discovered app
- Shows live build output (last 100 lines)
- Progress indicators
- Error detection and reporting

**Tauri Support**
- Separate "Compile" option for Tauri apps
- Detects `tauri:dev` and `tauri:build` scripts
- Handles Rust compilation output
- Shows build completion status

**Build States**
- Real-time output streaming
- Completion detection
- Auto-return to menu after build
- Error state handling

### 🌐 Deploy Applications

**Vercel Deployment**
- Deploy web app (staging/production)
- Deploy docs app (staging/production)
- Live deployment progress
- Output streaming during deploy

**Deployment Options**
- Staging deployment (`vercel deploy`)
- Production deployment (`vercel deploy --prod`)
- Per-app deployment paths
- Status feedback

### 🛑 Process Management

**Kill Dev Processes**
- Kills processes on common ports:
  - `42069` (Next.js default)
  - `6969` (alternative)
  - `1420` (alternative)
- Kills processes matching "tauri dev"
- Uses `lsof` and `pkill` commands
- Safe error handling

**Port Management**
- Checks if ports are in use
- Identifies blocking processes
- Clean process termination
