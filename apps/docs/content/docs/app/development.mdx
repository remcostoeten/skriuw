# Development Guide

This guide covers development workflows, coding standards, and best practices for building Notys.

## Development Workflow

### Daily Development

1. **Start the app:**
   ```bash
   cd apps/web
   bun run dev
   ```

2. **Make changes** - Hot reload handles most updates

3. **Test manually** - No automated tests yet; perform smoke tests

4. **Commit changes:**
   ```bash
   git add .
   git commit -m "feat: add task priority filter"
   ```

### Using SK for Development

SK provides a streamlined workflow:

```bash
# Launch interactive menu
bun run sk

# Available options:
# - Run Skriuw App (port 42069)
# - Run Docs App (port 3000)
# - Build Apps
# - Deploy to Vercel
```

**Hotkeys while running:**
- `O` - Open in browser
- `C` - Open in VS Code
- `R` - Restart process
- `S` - Stop process
- `I` - Install packages
- `Q` - Quit

See [SK documentation](/docs/cli) for full details.

## Project Conventions

### Coding Style

#### TypeScript

**File naming:**
- React components: `PascalCase.tsx`
- Hooks: `use-kebab-case.ts`
- Utilities: `kebab-case.ts`
- Types: `PascalCase.ts` or `kebab-case.ts`

**Import order:**
```typescript
// 1. External dependencies
import { useState } from 'react';
import { useQuery } from '@instantdb/react';

// 2. Internal absolute imports
import { Button } from '@/components/ui/button';
import { useGetNotes } from '@/modules/notes/api/queries/get-notes';

// 3. Relative imports
import { NoteEditor } from './note-editor';
import { formatDate } from '../utils';

// 4. Types
import type { Note } from '@/api/db/schema';
```

**Prefer named exports:**
```typescript
// ✓ Good
export function useGetNotes() { ... }

// ✗ Avoid
export default function useGetNotes() { ... }
```

**Use function declarations for components:**
```typescript
// ✓ Good
export function NoteEditor({ note }: Props) {
  return <div>...</div>;
}

// ✗ Avoid
export const NoteEditor: React.FC<Props> = ({ note }) => {
  return <div>...</div>;
};
```

#### Shared Type Utilities

- Reuse the shared primitives defined in `apps/web/src/types/global.d.ts` (and re-exported from `@/types`).
  - `UUID`, `Identifier`, `Nullable`, `Optional`, `Positionable`, `Timestamps`, `BaseEntity`, and `Children`.
- Avoid creating new union types like `string | null` or `number | undefined`; prefer the provided aliases (`Nullable<T>`, `Optional<T>`).
- When persisting data, call `withTimestamps` (or `createTimestamp`) from `@/shared/utilities/timestamps` instead of `Date.now()` inline.
- Keep `position` semantics aligned with the `Positionable` type and ensure new sortable entities implement it.
- If a new shared type is required, add it to `global.d.ts`, re-export it from `@/types`, and update this section alongside the `.cursor` shared type rule.

#### Formatting

**Prettier config:**
- Single quotes
- 4-space indentation
- No semicolons
- Trailing commas (ES5)

**Format on save:**
```bash
bun run format
```

**Check formatting:**
```bash
bun run format:check
```

#### Linting

**Run ESLint:**
```bash
bun run lint
```

**Auto-fix:**
```bash
bun run lint:fix
```

## Module Structure

### Creating a New Module

Example: Adding a "Labels" feature

```bash
apps/web/src/modules/labels/
├── api/
│   ├── queries/
│   │   ├── get-labels.ts
│   │   └── get-label.ts
│   └── mutations/
│       ├── create.ts
│       ├── update.ts
│       └── destroy.ts
├── components/          # Optional
│   └── label-picker.tsx
└── index.ts            # Public exports
```

### 1. Define Schema

Add to `src/api/db/schema.ts`:

```typescript
export const schema = i.graph(
  {
    // ... existing entities
    labels: i.entity({
      name: i.string(),
      color: i.string(),
      createdAt: i.number().indexed(),
    }),
  },
  {
    // ... existing relations
    noteLabels: {
      forward: { on: 'notes', has: 'many', label: 'labels' },
      reverse: { on: 'labels', has: 'many', label: 'notes' },
    },
  }
);
```

### 2. Create Query Hook

`modules/labels/api/queries/get-labels.ts`:

```typescript
import { createQueryHook } from '@/hooks/core';
import type { Label } from '@/api/db/schema';

const useLabelsQuery = createQueryHook(
  () => ({
    labels: {
      $: { order: { name: 'asc' } },
    },
  }),
  {
    select: (raw) => (raw?.labels as Label[]) ?? [],
    initialData: [] as Label[],
  }
);

export function useGetLabels() {
  const { data, isLoading, error } = useLabelsQuery();
  return { labels: data, isLoading, error };
}
```

### 3. Create Mutation Hook

`modules/labels/api/mutations/create.ts`:

```typescript
import { useCreate, useMutation } from '@/hooks/core';
import { generateId } from 'utils';

type CreateLabelInput = {
  name: string;
  color: string;
};

export function useCreateLabel() {
  const { create } = useCreate('labels');
  
  const { mutate, isLoading, error } = useMutation(
    async (input: CreateLabelInput) => {
      const id = generateId();
      const now = Date.now();
      
      await create(id, {
        ...input,
        createdAt: now,
      });
      
      return { id, ...input };
    },
    {
      errorContext: 'Creating label',
    }
  );

  return { createLabel: mutate, isLoading, error };
}
```

### 4. Create Component

`modules/labels/components/label-picker.tsx`:

```typescript
'use client';

import { useGetLabels } from '../api/queries/get-labels';
import { useCreateLabel } from '../api/mutations/create';

export function LabelPicker({ noteId }: { noteId: string }) {
  const { labels, isLoading } = useGetLabels();
  const { createLabel } = useCreateLabel();

  if (isLoading) return <div>Loading...</div>;

  return (
    <div>
      {labels.map((label) => (
        <button key={label.id}>{label.name}</button>
      ))}
      <button onClick={() => createLabel({ name: 'New', color: '#000' })}>
        Add Label
      </button>
    </div>
  );
}
```

### 5. Export from Module

`modules/labels/index.ts`:

```typescript
// Queries
export { useGetLabels } from './api/queries/get-labels';

// Mutations
export { useCreateLabel } from './api/mutations/create';
export { useUpdateLabel } from './api/mutations/update';
export { useDestroyLabel } from './api/mutations/destroy';

// Components (if applicable)
export { LabelPicker } from './components/label-picker';
```

## Common Patterns

### Pattern 1: Conditional Queries

Disable queries until data is ready:

```typescript
const useNoteQuery = createQueryHook(
  (noteId: string) => ({
    notes: {
      $: { where: { id: noteId } },
    },
  }),
  {
    select: (raw) => raw?.notes?.[0] ?? null,
    enabled: (noteId) => !!noteId,  // Only run if noteId exists
  }
);
```

### Pattern 2: Optimistic Updates

Update UI immediately, sync in background:

```typescript
export function useToggleTaskComplete() {
  const { update } = useUpdate('tasks');

  const { mutate } = useMutation(async ({ id, completed }: Props) => {
    // Updates local state immediately
    await update(id, { 
      completed,
      updatedAt: Date.now(),
    });
  });

  return { toggleComplete: mutate };
}
```

### Pattern 3: Derived State

Compute values from queries:

```typescript
export function useGetPinnedNotes() {
  const { notes, isLoading } = useGetNotes();

  const pinnedNotes = useMemo(
    () => notes.filter(note => note.pinned),
    [notes]
  );

  return { pinnedNotes, isLoading };
}
```

### Pattern 4: Compound Hooks

Combine multiple operations:

```typescript
export function useNoteOperations(noteId: string) {
  const { note, isLoading } = useGetNote(noteId);
  const { updateNote } = useUpdateNote();
  const { destroyNote } = useDestroyNote();
  const { duplicateNote } = useDuplicateNote();

  return {
    note,
    isLoading,
    updateNote,
    destroyNote,
    duplicateNote,
  };
}
```

## Testing Strategy

<Callout type="warning">
  Automated tests are not yet implemented. Manual testing is required.
</Callout>

### Manual Testing Checklist

For each feature:

1. **Happy path** - Normal usage works
2. **Edge cases** - Empty states, max values, special characters
3. **Error handling** - Network failures, validation errors
4. **Offline mode** - Works without internet
5. **Sync** - Changes sync across devices
6. **Performance** - No lag with 100+ items

### Future Test Plan

When adding automated tests:

```
tests/
├── unit/
│   ├── hooks/
│   └── utilities/
├── integration/
│   └── modules/
└── e2e/
    └── workflows/
```

**Recommended tools:**
- **Vitest** - Unit/integration tests
- **Playwright** - E2e tests
- **Testing Library** - Component tests

## Performance Tips

### 1. Memoize Expensive Computations

```typescript
const sortedNotes = useMemo(
  () => notes.sort((a, b) => a.position - b.position),
  [notes]
);
```

### 2. Debounce Inputs

```typescript
import { useDebouncedCallback } from 'use-debounce';

const debouncedSave = useDebouncedCallback(
  (content) => updateNote({ id, content }),
  500
);
```

### 3. Lazy Load Components

```typescript
import dynamic from 'next/dynamic';

const SettingsDialog = dynamic(() => import('./settings-dialog'), {
  loading: () => <Spinner />,
});
```

### 4. Index Frequently Queried Fields

```typescript
// In schema
createdAt: i.number().indexed(),
```

## Troubleshooting

### Hot Reload Not Working

1. Restart dev server
2. Clear `.next` cache: `rm -rf .next`
3. Check for syntax errors in console

### Skriuw Sync Issues

1. Check network tab for failed requests
2. Verify `NEXT_PUBLIC_INSTANT_APP_ID` in `.env`
3. Check Skriuw dashboard for outages

### Tauri Build Fails

**Linux:** Install missing dependencies (see [Setup Guide](/docs/app/setup))

**macOS:** Run `xcode-select --install`

**Windows:** Ensure Visual Studio Build Tools are installed

## Contributing

### Branch Naming

- `feat/feature-name` - New features
- `fix/bug-description` - Bug fixes
- `chore/task-name` - Maintenance tasks
- `docs/update-topic` - Documentation

### Commit Messages

Follow conventional commits:

```
feat: add task priority filter
fix: resolve note editor crash on empty content
chore: update dependencies
docs: improve setup instructions
```

### Pull Request Process

1. Create feature branch
2. Make changes following conventions
3. Test thoroughly (manual smoke tests)
4. Update documentation if needed
5. Commit with clear messages
6. Push and create PR
7. Address review feedback
