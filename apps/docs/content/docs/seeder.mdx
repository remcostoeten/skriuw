# Seeder Tool

The Seeder tool (`tools/seeder`) provides utilities for populating your Skriuw database with initial data, test data, and development fixtures.

## Overview

The seeder tool helps you:

- Populate databases with initial data
- Create test fixtures for development
- Seed sample content for demos
- Reset databases to known states

## Location

The seeder tool is located at `tools/seeder/`.

## Usage

### Running Seeders

From the project root:

```bash
cd tools/seeder
bun run start
```

Or use the CLI Manager (if integrated):

```bash
bun run cli
# Select seeder options (if available)
```

### Available Seeders

Check `tools/seeder/src/index.ts` for available seeders:

- Shortcuts note seeder
- Sample notes
- Test data
- Development fixtures

## Configuration

Seeder configuration is typically in:

- `tools/seeder/src/index.ts` - Main seeder logic
- Environment variables for database connection
- Seed data files in `tools/seeder/src/`

## Database Setup

Before seeding, ensure your database is set up:

1. **InstantDB**: Configure app ID and API key
2. **Environment**: Set `.env` variables
3. **Connection**: Verify database connectivity

## Development

### Adding New Seeders

1. Create a new seeder function in `tools/seeder/src/`
2. Export it from `index.ts`
3. Add to the seeder menu
4. Test with development database

### Seed Data Format

Seeders typically create:

- Notes with content
- Folders/projects
- User settings
- Shortcuts
- Tasks (if applicable)

## Best Practices

1. **Use Development Database** - Never seed production
2. **Idempotent** - Seeders should be safe to run multiple times
3. **Clear First** - Optionally clear existing data before seeding
4. **Documentation** - Document what each seeder creates

## Troubleshooting

### Seeder Fails

1. Check database connection
2. Verify environment variables
3. Check InstantDB schema matches
4. Review error messages

### Data Not Appearing

1. Check database connection
2. Verify seeder ran successfully
3. Check app is connected to same database
4. Refresh the app
