# Troubleshooting

Solutions to common problems you might encounter while using SK.

## Installation Issues

### "bun: command not found"

**Problem**: Bun is not installed on your system.

**Solution**:
```bash
# Install Bun
curl -fsSL https://bun.sh/install | bash

# Verify installation
bun --version
```

### Dependencies Not Installing

**Problem**: Running `bun install` fails or hangs.

**Solution**:
```bash
# Clear bun cache
rm -rf ~/.bun/install/cache

# Try again
bun install

# If still failing, check your internet connection
# or try with npm/pnpm instead
npm install
```

## Startup Issues

### CLI Won't Launch

**Problem**: `bun run cli` shows error or hangs.

**Solution**:

1. Check you're in the project root:
   ```bash
   pwd
   # Should show: /path/to/skriuw
   ```

2. Verify CLI is installed:
   ```bash
   ls tools/sk/
   ```

3. Rebuild the CLI:
   ```bash
   cd tools/sk
   bun install
   bun run build
   ```

4. Try running directly:
   ```bash
   cd tools/sk
   bun run dev
   ```

### "Cannot find module"

**Problem**: TypeScript or JavaScript module errors.

**Solution**:
```bash
# Reinstall dependencies
rm -rf node_modules
rm -rf tools/*/node_modules
bun install

# Rebuild TypeScript
cd tools/cli-manager
bun run build
```

## App Running Issues

### Port Already in Use

**Problem**: App fails to start with "EADDRINUSE" error.

**Solution**:

1. Find what's using the port:
   ```bash
   # For port 42069 (Tauri app)
   lsof -i :42069
   
   # For port 3000 (Docs)
   lsof -i :3000
   ```

2. Kill the process:
   ```bash
   kill -9 [PID]
   ```

3. Or change the port in `tools/sk/src/config.ts`:
   ```typescript
   port: 42070  // Changed from 42069
   ```

### App Won't Start

**Problem**: Spinner runs forever, no "Running" status.

**Solution**:

1. Stop the CLI (Ctrl+C)

2. Try starting the app manually:
   ```bash
   cd apps/instantdb
   bun run dev
   ```

3. Check for errors:
   - Missing dependencies
   - Environment variables
   - Configuration issues

4. Install missing dependencies:
   ```bash
   bun install
   ```

5. Check environment variables:
   ```bash
   # Create .env if missing
   cp .env.example .env
   # Edit with your values
   ```

### App Starts but Doesn't Respond

**Problem**: App shows "Running" but `http://localhost:port` doesn't work.

**Solution**:

1. Check if the process is actually running:
   ```bash
   ps aux | grep node
   ```

2. Check app logs manually:
   ```bash
   cd apps/[app-name]
   bun run dev
   ```

3. Verify port in browser matches config

4. Try restarting:
   - Select "Manage Running Apps"
   - Press `R` to restart

## Process Management Issues

### Hotkeys Not Working

**Problem**: Pressing O, C, R, S, I does nothing.

**Solution**:

1. Ensure you're in "Manage Running Apps" menu
2. Select an app first (use arrow keys + Enter)
3. Then use hotkeys
4. Check terminal compatibility:
   ```bash
   # Some terminals don't support raw mode
   # Try a different terminal (iTerm2, Alacritty, etc.)
   ```

### Can't Stop Apps

**Problem**: Apps keep running after exiting CLI.

**Solution**:

1. Find running Node processes:
   ```bash
   ps aux | grep node
   ```

2. Kill them:
   ```bash
   killall node
   # or
   kill -9 [PID]
   ```

3. Use proper exit:
   - Press Ctrl+C or
   - Select "Exit" from menu

### Multiple Instances Running

**Problem**: Same app running multiple times on different ports.

**Solution**:

1. Check all Node processes:
   ```bash
   ps aux | grep node
   ```

2. Stop all instances:
   ```bash
   killall node
   ```

3. Start CLI fresh and use "Run All Apps" instead of starting individually multiple times

## Build Issues

### Build Fails

**Problem**: "Build [App Name]" shows errors.

**Solution**:

1. Read error messages carefully
2. Common issues:

   **TypeScript Errors**:
   ```bash
   # Check for type errors
   cd apps/[app-name]
   bun run build
   # Fix reported errors
   ```

   **Missing Dependencies**:
   ```bash
   bun install
   ```

   **Environment Variables**:
   ```bash
   # Check if .env is needed
   cp .env.example .env
   ```

3. Try cleaning build artifacts:
   ```bash
   cd apps/[app-name]
   rm -rf .next dist out
   bun run build
   ```

### Build Takes Too Long

**Problem**: Builds never complete or take 10+ minutes.

**Solution**:

1. Check system resources:
   ```bash
   top
   ```

2. Close other applications

3. Try building individually instead of "Build All":
   - Select "Build [specific app]"

4. Check for infinite loops in build scripts

## Deployment Issues

### "vercel: command not found"

**Problem**: Deployment features don't work.

**Solution**:
```bash
# Install Vercel CLI
npm i -g vercel

# Login
vercel login

# Try again
```

### Deployment Fails

**Problem**: "Deploy to Staging/Production" shows errors.

**Solution**:

1. Ensure Vercel is logged in:
   ```bash
   vercel whoami
   ```

2. Check project is linked:
   ```bash
   cd apps/[app-name]
   vercel link
   ```

3. Build locally first:
   - Select "Build [App Name]"
   - Ensure it succeeds
   - Then deploy

4. Check Vercel dashboard for detailed errors

## Configuration Issues

### Changes Not Applying

**Problem**: Modified `config.ts` but changes don't show.

**Solution**:

1. Rebuild the CLI:
   ```bash
   cd tools/sk
   bun run build
   ```

2. Restart the CLI

3. For live changes during development:
   ```bash
   cd tools/sk
   bun run dev
   ```

### Syntax Errors in Config

**Problem**: Config file has errors, CLI won't start.

**Solution**:

1. Check TypeScript compilation:
   ```bash
   cd tools/sk
   bun run build
   # Read error messages
   ```

2. Common mistakes:
   - Missing commas in arrays
   - Unclosed strings
   - Wrong property names

3. Validate against the interface:
   ```typescript
   export interface AppConfig {
     name: string;
     displayName: string;
     path: string;
     dev: string;
     build: string;
     port: number;
     color: string;
   }
   ```

## Terminal Issues

### Colors Not Showing

**Problem**: Output is plain text, no colors.

**Solution**:

1. Check terminal supports ANSI colors
2. Try a modern terminal:
   - iTerm2 (macOS)
   - Terminal.app (macOS)
   - Alacritty (Linux/macOS)
   - GNOME Terminal (Linux)

3. Check if colors are disabled:
   ```bash
   # Force colors
   export FORCE_COLOR=1
   bun run cli
   ```

### Menu Navigation Broken

**Problem**: Arrow keys print characters instead of navigating.

**Solution**:

1. Try a different terminal emulator
2. Check SSH connection (some don't support interactive mode)
3. Run locally instead of over SSH
4. Use number keys if available

### CLI Hangs on Input

**Problem**: CLI freezes waiting for input.

**Solution**:

1. Press Ctrl+C to force exit
2. Check if stdin is properly configured
3. Not in a pipe or redirect:
   ```bash
   # Bad:
   echo "" | bun run cli
   
   # Good:
   bun run cli
   ```

## Performance Issues

### Slow Startup

**Problem**: CLI takes several seconds to show menu.

**Solution**:

1. Check if TypeScript is compiled:
   ```bash
   ls tools/sk/dist/
   # Should see index.js
   ```

2. Use compiled version instead of tsx

3. Reduce number of apps in config

4. Check system resources

### High CPU Usage

**Problem**: CLI or apps consuming lots of CPU.

**Solution**:

1. Check for infinite loops in app code

2. Limit number of running apps

3. Check for file watchers:
   ```bash
   # Increase file watcher limit on Linux
   echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
   sudo sysctl -p
   ```

## Git Issues

### "Open Repository" Fails

**Problem**: Repository doesn't open in browser.

**Solution**:

1. Check Git remote exists:
   ```bash
   git remote -v
   ```

2. Ensure remote is HTTPS (not SSH):
   ```bash
   # Should show https://github.com/...
   # Not git@github.com:...
   ```

3. Add HTTPS remote if needed:
   ```bash
   git remote set-url origin https://github.com/user/repo.git
   ```

## Still Having Issues?

### Get Help

1. **Check Logs**: Run apps manually to see full output
2. **Verify Setup**: Ensure all prerequisites are installed
3. **Update Dependencies**: `bun update` in project root
4. **Clean Install**: Delete node_modules and reinstall
5. **Check Documentation**: Review [Configuration](/docs/cli/configuration) and [Usage](/docs/cli/usage)

### Debug Mode

Run with verbose output:

```bash
cd tools/cli-manager
NODE_ENV=development bun run dev
```

### Report a Bug

If you've found a bug:

1. Note your environment:
   - OS and version
   - Bun version (`bun --version`)
   - Node version (`node --version`)

2. Steps to reproduce

3. Error messages (copy full output)

4. Create an issue in the repository

## Prevention Tips

### Before Starting Development

- [ ] Run `bun install` in project root
- [ ] Check ports are available
- [ ] Verify environment variables set
- [ ] Ensure Git remote is configured

### Regular Maintenance

- [ ] Update dependencies monthly
- [ ] Clear caches if experiencing issues
- [ ] Rebuild CLI after config changes
- [ ] Check for app updates

### Best Practices

1. **One CLI instance** - Don't run multiple CLIs
2. **Clean exits** - Always use Exit or Ctrl+C
3. **Monitor ports** - Track what's running where
4. **Regular restarts** - Restart apps if behaving oddly
5. **Stay updated** - Keep dependencies current
